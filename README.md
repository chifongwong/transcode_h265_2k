# transcode_h264.sh

一個 macOS 上用 `ffmpeg` 把影片轉碼成 **H.264** 的小工具。輸入 mkv 或 mp4，輸出到原檔同一個資料夾，並自動處理解析度與（選用的）起始時間點。預設把過大的影片降到寬 1280（720p），適合手機串流。

## 功能

- 影像轉 **H.264**（`libx264`），音訊轉 **AAC**，輸出 `.mp4` 容器，相容性最佳。
- **自動偵測解析度**：來源寬度超過 1280px（720p）時，等比例降到寬 1280；低於此門檻則不縮放、直接繼續轉碼。
- **選用的起始時間點**：可從指定時間開始轉碼。
- 輸出檔為 `原檔名_h264.mp4`，存放於**原檔同一資料夾**。
- 內建保護：找不到檔案、無法偵測寬度、或輸出檔已存在時，會停止並提示，不會覆蓋既有檔案。

## 需求

- macOS
- [ffmpeg](https://ffmpeg.org/)（含 `ffprobe`，一般會一起安裝）
- 相容 macOS 內建的 **bash 3.2**，不需另裝新版 bash（詳見「疑難排解」）

安裝：

```bash
brew install ffmpeg
```

## 安裝腳本

把腳本存成 `transcode_h264.sh`，並加上執行權限：

```bash
chmod +x transcode_h264.sh
```

## 用法

```bash
./transcode_h264.sh <影片路徑> [開始時間]
```

| 參數 | 必填 | 說明 |
| --- | --- | --- |
| `影片路徑` | 是 | 來源影片檔（mkv、mp4 等）|
| `開始時間` | 否 | 從此時間點開始轉碼。格式為秒數（如 `90`）或 `時:分:秒`（如 `00:01:30`）|

### 範例

```bash
# 從頭開始轉碼
./transcode_h264.sh "/path/to/影片.mkv"

# 從第 90 秒開始
./transcode_h264.sh "/path/to/影片.mp4" 90

# 從 1 分 30 秒開始
./transcode_h264.sh "/path/to/影片.mkv" 00:01:30
```

輸入 `/path/to/影片.mkv` 會產生 `/path/to/影片_h264.mp4`。

## 參數說明（方便自行調整）

| 參數 | 目前值 | 說明 |
| --- | --- | --- |
| `-crf` | `20` | 畫質。數字越小畫質越好、檔案越大，常用範圍 18–23，20 是不錯的甜蜜點。 |
| `-preset` | `medium` | 編碼速度 vs 壓縮效率。想更快可用 `fast`／`veryfast`；想壓更小可用 `slow`。 |
| `-pix_fmt` | `yuv420p` | 確保 QuickTime、iOS 等能正常播放。 |
| `-b:a` | `192k` | AAC 音訊位元率。 |
| `-movflags` | `+faststart` | 把索引移到檔頭，網路串流或線上播放時可邊下載邊播。 |
| `MAX_WIDTH` | `1280` | 縮放門檻（寬度）。超過才降到寬 1280（720p，手機串流甜蜜點）。 |

## 技術重點

- **縮放**用 `scale=1280:-2`，`-2` 表示高度依比例自動計算並取偶數（H.264 的 yuv420p 要求寬高皆為偶數）。
- **起始時間**用 `-ss`，且置於 `-i` **之前**（input seeking）。因為本工具會重新編碼，這個位置既快又精準。
- 縮放門檻以**寬度**判斷，適合一般 16:9 橫式素材；超寬或直式影片可能需改用高度或像素總數判斷。
- 若要改門檻，只需改 `MAX_WIDTH` 一個值：`1024`（最省流量）、`1280`（720p，手機串流甜蜜點，預設）、`1920`（1080p）、`2560`（2K，手機串流幾乎用不到）。
- 縮放與起始時間參數用陣列儲存，並以 `"${arr[@]+"${arr[@]}"}"` 展開。這樣在 `set -u` 下，空陣列（不縮放或不指定起始時間時）會展開成零個參數，而非報錯退出，也不會塞進一個空參數把 ffmpeg 弄壞。

## 常見調整

**不重新編碼音訊（來源已是 AAC 時）**：把音訊那行改為 `-c:a copy`，可省時間並保留原音質。mp4 轉 mp4 時音訊通常已是 AAC，用 `copy` 多半安全；但 mkv 常見的 FLAC／DTS 放進 mp4 相容性差，仍建議轉 AAC。

**控制結束時間或長度**：可再加 `-t`（持續長度，如 `-t 60` 截 60 秒）或 `-to`（截到某時間點）。

## 疑難排解

**來源不需縮放時本來會中途停掉 / 出現 `unbound variable` 錯誤**

macOS 內建的是 bash 3.2。腳本開頭有 `set -u`（遇到未定義變數就退出），而 bash 3.2 在 `set -u` 下展開空陣列（如來源不需縮放時的 `scale_args`）會被當成未定義變數而報錯退出。因此舊寫法會在「來源寬度已在門檻內、不需縮放」的情況停掉。

本腳本已用 `"${arr[@]+"${arr[@]}"}"` 的展開方式修正，空陣列會安全地展開成零個參數，不再退出。若你手上是舊版腳本，兩種修法擇一即可：

- 改用上述安全展開寫法（本專案採用，免安裝任何東西）。
- 或改用 Homebrew 安裝的新版 bash：`brew install bash`，並把腳本首行改成 `#!/opt/homebrew/bin/bash`。
